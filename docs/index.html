Security by Design

Every feature built with security as the foundation

üîê WebAuthn (FIDO2)

Passwordless authentication using public key cryptography. Resistant to phishing, credential stuffing, and man-in-the-middle attacks.

üõ°Ô∏è Content Security Policy

Strict CSP headers prevent XSS attacks. No inline scripts, no eval(), no unsafe practices.

üîí CSRF Protection

All state-changing operations require CSRF tokens. Prevents cross-site request forgery attacks.

‚öõÔ∏è Atomic Operations

State mutations use file locking and atomic writes. No race conditions, no data corruption.

üö´ Path Traversal Protection

Strict path validation prevents directory traversal attacks. All file operations are sandboxed.

üì¶ Self-Hosted Dependencies

All JavaScript dependencies served from your server. No CDN trust, no supply chain attacks.

üîç Input Validation

Comprehensive input validation and sanitization. Format validation for UUIDs, PINs, and user data.

üöÄ Secure Defaults

Localhost auto-authenticated, remote requires WebAuthn. TLS with self-signed certs for LAN access.

86 security improvements completed in February 2026, including request body DoS protection, header trust removal, and environment variable filtering.

Quick Start with CLI

1

Start Katulong

katulong start

Starts daemon and server on ports 3001 (HTTP) and 3002 (HTTPS)

2

Open in Browser

katulong open

Or visit http://localhost:3001 manually. Follow the WebAuthn registration flow to create your first passkey.

3

Manage Services

katulong status # Check if running katulong logs # View output katulong stop # Stop services

Run katulong --help to see all available commands

4

Pair Additional Devices

From Settings ‚Üí "Pair New Device", scan the QR code on your mobile device and enter the PIN to pair securely.
